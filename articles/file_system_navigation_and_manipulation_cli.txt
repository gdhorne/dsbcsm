[RStudio][2014] provides seamless integration with the file system to navigate and
manipulate files, the version control system to manage and synchronise local
and remote source code repositories, and the statistical computation and
software development envrionment. However, becoming comfortable with the
command line will improve our productivity as data analysis projects increase
 in sophistication and work-flow automation arises.

### Bourne Again SHell (BASH) Commands

[BASH][2012] is an interactive command shell for *nix-like operating systems.

Let's start exploring basic file system navigation and manipulation at the
command-line. Open a web browser tab and type 'http://127.0.0.1:8000' to access
a terminal session. The user name and password are provided in the
[Toolbox for Tools](../toolbox_for_tools/index.html) tutorial. 
By learning a few basic commands to navigate and manipulate the file system we
will feel more at ease and understand what is going on behind the scenes when
using RStudio.

### Hands-on Practise

For each example type the commands to the right of the command prompt ($) to 
interactively follow along these examples. Take your time working through the 
commands until you fully understand why each command produces the observed 
results.

<br/> 
Example 1:

Determine the current working directory.

	$ pwd
	/home/dst

<br/>
Example 2:

List the file and subdirectory names in the current working directory.

	$ ls
	bin  data  tmp  www

<br/>
Example 3:

Create a subdirectory named 'test' in the current directory.

	$ mkdir test
	$ cd test
	$ pwd
	/home/dst/test

<br/>
Example 4:

Create subdirectories named '1', '2', '3', and '4' in the current directory.

	$ mkdir {1,2,3,4}

<br/>
Example 5:

List the files and subdirectory names in the current directory.

	$ ls
	1  2  3  4

<br/>
Example 6:

Create some empty files and some files with content.

	$ touch 1/file01.txt 2/file01.txt
	$ echo "Hello World!" > ./1/file02.txt
	$ echo "To be or not to be" >> ./1/file03.txt
	$ echo "The quality of mercy is not strained" > ./2/file02.txt

<br/>
Example 7:

List of files in the current directory and all subdirectories in the current
directory.

	$ ls -lR
	.:
	total 16
	drwxrwxr-x 2 dst dst 4096 Jan 11 05:55 1
	drwxrwxr-x 2 dst dst 4096 Jan 11 05:55 2
	drwxrwxr-x 2 dst dst 4096 Jan 11 05:54 3
	drwxrwxr-x 2 dst dst 4096 Jan 11 05:54 4
	./1:
	total 8
	-rw-rw-r-- 1 dst dst  0 Jan 11 05:54 file01.txt
	-rw-rw-r-- 1 dst dst 13 Jan 11 05:54 file02.txt
	-rw-rw-r-- 1 dst dst 19 Jan 11 05:55 file03.txt
	./2:
	total 4
	-rw-rw-r-- 1 dst dst  0 Jan 11 05:54 file01.txt
	-rw-rw-r-- 1 dst dst 37 Jan 11 05:55 file02.txt
	./3:
	total 0
	./4:
	total 0

<br/>
Example 8:

Display the content of file02.txt in subdirectory ~/data/test/1.

	$ cat ~/test/1/file02.txt
	Hello World!

<br/>
Example 9:

Copy ~/data/test/1/file03.txt to the ${HOME} directory.

	$ cp 1/file03.txt ${HOME}
	$ ls -l ${HOME}
	total 16
	drwxr-xr-x  2 dst dst 4096 Jan 01 14:01 bin
	drwxrwxr-x 14 dst dst 4096 Jan 01 14:01 data
	-rw-rw-r--  1 dst dst   19 Jan 11 05:00 file03.txt
	drwxr-xr-x  2 dst dst 4096 Jan 01 14:01 tmp

<br/>
Example 10:

Move 1/file03.txt to the ${HOME} directory and rename it file04.txt.

	$ mv 1/file03.txt ~/file04.txt
	$ ls -l 1/*.txt
	-rw-rw-r-- 1 dst dst  0 Jan 11 05:54 1/file01.txt
	-rw-rw-r-- 1 dst dst 13 Jan 11 05:54 1/file02.txt	
	$ ls -l ${HOME}
	total 20
	drwxr-xr-x  2 dst dst 4096 Jan 01 14:01 bin
	drwxrwxr-x 14 dst dst 4096 Jan 11 05:55 data
	-rw-rw-r--  1 dst dst   19 Jan 11 05:00 file03.txt
	-rw-rw-r--  1 dst dst   19 Jan 11 05:55 file04.txt
	drwxr-xr-x  2 dst dst 4096 Jan 01 14:01 tmp

<br/>
Example 11:

Remove all directories and files created for this tutorial.

	$ rm ${HOME}/*.txt
	$ rm -r ${HOME}/data/test && cd ${HOME}

<br/>
Example 12:

End the terminal session.

	$ logout
